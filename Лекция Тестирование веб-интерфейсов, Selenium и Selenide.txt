Клиентом при клеинт серверном взаимодействии может быть десктоп приложение ,мобильное приложение
веб браузер.

Когда мы переходим по какому то url адрессу клиент генерирует и  отправляет на сервер http запрос,
сервер обрабатывает информацию и возвращает нам информацию в виде json или в виде html документа.
После получаения ответа клиент обрабатывает полученные данные так же может запросить дополнительные ресурсы для отображения страницы js,css,png
и в итоге клиент получает итоговую страницу с которой пользователь может взаимодействовать.

Http - это текстовый протокол передачи данных. В этом виде данных клиент формирует запрос для обращения к серверу(подчиняющиеся правилам протокола).
На запрос сервер отвечает сообщением так же подчиняющийся определнным правилам.

Загружая страницы по url адрессу, мы получаем html код который браузер отображает в виде страницы,и объекты этой страницы являются html элементами 
с которыми мы можем и должны взаимодействовать.(Например: заполнять поля для ввода , нажимать на кнопки , отправлять форму , ставить галочки в чекбоксах,проверять наличе правильного текста)

Selenide.

Искать элементы на страницы мы можем при помощи их локаторов,локаторами могут быть теги ,классы и атрибуты

Если мы ищем по тегу, в поле поиска элементов вводим тег веб элемента 
например поиск по тегу input
так же можно искать веб элементы вложенные в несколько тегов 
например по поиску тега p в который вложен тег input 
p input - будут найдены все элемнты типа input вложенные в тег p
но такой поиск не будет уникальным для какого то определенного элемента

Так же можем искать элемент по атрибуту например 
input[type]
И так же можем искать по значению атрибут,(искав таким образом мы найдем уникальный веб элемент).
input[@placeholder="Введите ФИО"]

Поиск по классу осуществляется через точку и знаечние строки class
например 
.page-form__label

Поиск веб элемента в xpath отличается тем что перед тегом мы обязательно должны ставить два слэша 
например //input
если мы хотим найти дочерние элементы тегов тогда 
//p/input

так же при поиске атрибутов 
//input[@placeholder="Введите ФИО"]

В xpath есть возможность не привязываться к тегу , и заменить тег на звездочку *
например //*[@placeholder="Введите ФИО"]

Главное отличие в поиске элементов при помощи xpath в том что мы можем искать элементы по тексту, это отличает xpath от css селекторов.

Если мы хотим найти элемент на странице по содержанию текста тогда мы должны ввести
//a[text()="Аллерголог"]
или 
//*[text()="Аллерголог"]

Если мы хотим найти элемент по неполной части текста,используем
//a[contains(text()="Медицинский")]
или 
//*[contains(text()="Медицинский")]
так же можем искать по содержанию атрибута 
//*[contains(@placeholder,"Введите ФИО")]

Для навигации через строку поиска элементов необходимо использовать 
/..
это полезно тогда когда нам необходимо найти веб элемент но у него отсутствует уникальный локатор
